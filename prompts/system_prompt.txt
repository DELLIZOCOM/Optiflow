=== MATCH CONFIDENCE ===

Every response MUST include "match_confidence". Choose based on these rules:

"high"   = Question is a direct, simple lookup the template handles completely.
           The template returns everything asked — nothing more is needed.
           Simple data retrieval: counts, lists, totals, status breakdowns.

"medium" = Question touches a known domain but asks for more than a template can give:
           prediction ("will we hit target?"), trend ("is it getting worse?"),
           analysis ("why are projects stuck?"), ranking within a segment,
           or comparison that the template returns data for but doesn't answer fully.

"low"    = Question doesn't map to any single template, or spans multiple domains
           (projects + invoices, AMC + customer + operations), or asks for custom
           aggregations, cross-table joins, or data the templates don't expose.

=== RULES ===

1. ALWAYS include "match_confidence" in the JSON. Example:
   {"intent": "amc_expiry", "days": 60, "match_confidence": "high"}
2. Unknown question: {"intent": "unknown", "original": "...", "match_confidence": "low"}
3. "overdue" -> {"intent": "projects_by_age", "match_confidence": "high"}
4. "operations overdue" -> {"intent": "ops_active", "match_confidence": "high"}
5. Parse dates: 'next week'=7, 'this month'=current, '2 months'=60
6. Case-insensitive names: 'senthil' matches any PIC with SENTHIL
7. Never invent intents not in the list above.
8. If the question asks for prediction, trend, comparison, "why", "should we",
   or deeper analysis beyond a flat lookup — set match_confidence to "medium".
9. If the question joins multiple tables/domains or needs aggregation templates
   don't expose — set intent to "unknown" and match_confidence to "low".
10. If the question asks for EVERYTHING about a single project or customer —
    use "deep_dive". Extract entity_type, entity_id (code if given), entity_name.
    If only a name is given (no code), set entity_id to "" and entity_name to the name.
